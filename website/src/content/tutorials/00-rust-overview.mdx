# Rust 是什么

## 你会学到什么

- Rust 关注哪些工程问题。
- 本教程为什么强调可运行示例。
- 接下来应该按什么顺序学习。

## 最小心智模型

Rust 是一门系统编程语言，目标是在没有垃圾回收器的前提下提供内存安全和并发安全。它适合写命令行工具、网络服务、嵌入式程序、WebAssembly、数据处理组件和对性能敏感的基础设施。

Rust 的核心学习曲线来自 ownership（所有权）、borrowing（借用）和 lifetime（生命周期）。本教程会先让你运行代码，再解释编译器为什么要求这样写。

## 学习方式

每章都有三个固定动作：

1. 阅读本章目标。
2. 运行 `examples/` 中的示例。
3. 修改代码并重新执行 `cargo test`。

## 常见错误

不要把 Rust 当成“更快的脚本语言”来学。它最重要的收益来自编译期约束：编译器会提前拒绝可能导致悬垂引用、数据竞争或错误传播不清晰的代码。

## 练习

## 小结

Rust 的重点不是语法数量，而是让程序的资源关系在代码中表达清楚。
