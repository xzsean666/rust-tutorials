# 生命周期入门

## 你会学到什么

- 生命周期标注 `'a` 描述引用能存活多久，**不改变**值实际的生存期。
- 当函数返回引用时，为什么借用检查器需要你显式标注。
- 持有引用的结构体必须带生命周期参数，以及如何为它写方法。
- 生命周期省略规则（elision）：多数时候你根本不用写 `'a`。
- `'static`：贯穿整个程序的生命周期。

## 最小示例

```rust
fn longest<'a>(left: &'a str, right: &'a str) -> &'a str {
    if left.len() >= right.len() { left } else { right }
}
```

## 代码讲解

下面对照本页底部自动嵌入的完整源码，逐个概念展开。

### 为什么 `longest` 需要 `'a`

`longest` 返回一个引用，但它在编译期**无法确定**返回的到底是 `left` 还是 `right`（取决于运行时长度）。借用检查器因此不知道该用哪个入参的存活期来约束返回值。

`<'a>` 声明一个生命周期参数；把两个入参和返回值都标成 `&'a str`，含义是：“返回的引用至少和两个入参中**存活更短**的那个一样久”。调用方据此就能保证：只要 `left`、`right` 都还活着，返回值就是安全的。注意示例 `main` 里我们故意让两个入参来自不同作用域，说明它们的生存期不必相同。

### 持有引用的结构体与方法

结构体若存引用，就必须带生命周期参数：

```rust
struct Excerpt<'a> {
    part: &'a str,
}
```

这表示 `Excerpt` 实例**不能比**它引用的字符串活得更久。`impl<'a> Excerpt<'a>` 里我们写了构造函数 `new` 和两个方法，演示如何在带生命周期的类型上实现行为。

### 生命周期省略规则

并非所有引用都要手写 `'a`。编译器内置了几条**省略规则**：

- 每个引用参数各得一个独立的生命周期；
- 若只有**一个**输入引用，输出引用就用它的生命周期；
- 若参数里有 `&self`（方法），输出引用一律跟随 `&self`。

所以示例里的 `first_word(s: &str) -> &str` 和方法 `part(&self) -> &str` 都不用写 `'a`——这也正是前几章我们从没见过生命周期标注的原因。`announce_and_return_part` 即使多带一个 `&str` 参数，因为有 `&self`，返回值依旧无需标注。

### `'static`

`'static` 表示引用在整个程序运行期间都有效。字符串字面量天生是 `&'static str`（被编进二进制、永不释放），所以 `motto()` 能安全地把它返回出来。

## 常见错误

返回一个指向函数内部局部变量的引用：

```rust
fn dangling() -> &String {
    let s = String::from("temp");
    &s // ❌ s 在函数结束时被释放
}
```

修复方式是返回所有权（`String`）而不是引用。

## 练习

- 给 `Excerpt` 加一个方法 `len`，返回 `part` 的字节长度（试试它需不需要写 `'a`）。
- 写一个函数 `first<'a>(items: &'a [String]) -> &'a str` 返回第一个元素的切片，并思考它能否靠省略规则去掉 `'a`。
- 把 `motto` 的返回类型从 `&'static str` 改成 `&str`，看看编译器报什么错，体会 `'static` 的含义。

## 小结

生命周期是借用检查器的“时间维度”，让返回引用和存引用的结构体都安全。多数时候靠省略规则自动推断。
